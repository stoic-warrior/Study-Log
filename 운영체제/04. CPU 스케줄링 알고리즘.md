> ready queue에 있는 스레드 중 어떤 것을 CPU에 올릴지 결정하는 정책

- **CPU 스케줄링의 목표**
  - **CPU 이용률(Utilization) 극대화**: CPU가 놀지 않게 한다
  - **처리량(Throughput) 증가**: 단위 시간당 처리되는 작업 수↑, 많은 프로세스 처리 
  - **대기 시간(Waiting Time) 최소화**: ready queue에서 기다리는 시간↓
  - **응답 시간(Response Time) 단축**: 사용자 요청에 대해 빠르게 반응해야 한다
# 비선점형 방식 (Non-preemptive)
> CPU를 한번 받으면 끝날 때까지 뺏기지 않는다
- CPU를 강제로 빼앗지 않아서 컨텍스트 스위칭 비용이 낮다
## FCFS (First Come First Served)
> 먼저 온 프로세스가 먼저 실행]
- 장점 : 구현 간단 
- 단점 : Convoy Effect
  - 실행시간이 긴 프로세스 때문에 짧은 프로세스들이 Ready Queue에서 기다리는 현상 
## SJF (Shortest Job First)
> 가장 짧은 작업 먼저 실행
- 실행시간을 미리 알 수 없으니 예측을 활용한다 (과거에 실행했던 시간을 토대로 추측)
- 장점 : 평균 대기 시간 최소
- 단점 : 기아 현상(starvation) : 실행시간이 긴 프로세스가 영원히 실행되지 않음 
## 우선순위 (Priority)
> 우선순위가 높은 프로세스 먼저 실행
- SJF의 starvation을 방지하기 위해, 오래된 작업의 우선순위를 높이는 방법(aging)을 사용할 수 있다
- 우선순위는 SJF뿐만 아니라, FCFS이나, 선점형이 될 수도 있다  
# 선점형 방식 (Preemptive)
> OS가 프로세스의 CPU를 뺏을 수 있다
- 장점
- 응답성이 좋음
- 인터랙티브 시스템에 적합
- 단점
- 컨텍스트 스위칭 많음
- 구현 복잡
## 라운드 로빈 (Round Robin, RR)
> 각 프로세스에 동일한 CPU시간 순환할당
- 현대 컴퓨터가 쓰는 방식
- 각 프로세스에 동일한 할당시간을 주고 그 안에 안끝나면 ready queue로 감
- 타임 슬라이스 크기
  - 너무 크면 FCFS랑 같아짐
  - 너무 작으면 컨텍스트 스위칭 비용이 커짐
- 일반적으로는 전체 작업 시간은 길어지고, 평균 응답 시간은 짧아짐
