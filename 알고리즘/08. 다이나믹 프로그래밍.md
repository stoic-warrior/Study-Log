# 중복되는 연산을 줄이자
- 한 번 해결한 문제는 다시 계산하지 않도록 메모리에 저장하여 연산 속도를 비약적으로 높이는 알고리즘 기법
## 피보나치 수열 
<img width="666" height="211" alt="image" src="https://github.com/user-attachments/assets/14be6bda-de15-4896-87b5-198250b17989" />

- 점화식 : $a_n = a_{n-1} + a_{n-2}$
  - $a_1 = 1, a_2 = 2 $
<img width="245" height="261" alt="image" src="https://github.com/user-attachments/assets/6cc19eaf-fa1a-42bf-a133-9d9d96243f31" />

- f(1)이나 f(2)를 만나면 호출 정지
- 점화식은 재귀함수를 사용하면 구현 쉽다
```
public static int fibo(int x) {
    // 종료 조건 (1 혹은 2일 때 1을 반환)
    if (x == 1 || x == 2) {
        return 1;
    }
    return fibo(x - 1) + fibo(x - 2);
}

```

- (P) 시간복잡도가 $O(2^N)$이다
<img width="580" height="328" alt="image" src="https://github.com/user-attachments/assets/2e635e67-5a63-4a1d-ba27-53eca0855c13" />

- f(6)의 호출 과정을 보면, 동일한 함수가 반복적으로 호출 되는 것을 알 수 있다
- 위 그림에서 f(3)만 봐도 3번 호출됐다
- (S) 이런 문제를 다이나믹 프로그래밍으로 해결한다
## 사용조건
1. 큰 문제를 작은 문제로 나눌 수 있다.
2. 작은 문제에서 구한 정답은 그것을 포함하는 큰 문제에서도 동일하다.
- **분할 정복(Divide and Conquer)과의 차이:** DP는 문제들이 서로 영향을 미칩니다(중복된 하위 문제). 반면 퀵 정렬 같은 분할 정복은 한 번 해결한 부분 문제가 반복되지 않는다 (피벗이 바뀐다)
## 탑다운 방식
> 큰 문제를 풀기 위해 작은 문제를 호출하는 방식 (재귀 함수 사용)
- 계산한 결과를 메모리에 저장해두고, 같은 문제를 만나면 계산없이 메모리에서 결과만 꺼내오는 방법
- **메모제이션 기법, 캐싱**이라고도 부른다
```
public static long fibo(int x) {
        // 종료 조건 (1 혹은 2일 때 1을 반환)
        if (x == 1 || x == 2) {
            return 1;
        }

        // 이미 계산한 적 있는 문제라면 그대로 반환
        if (d[x] != 0) {
            return d[x];
        }

        // 아직 계산하지 않은 문제라면 점화식에 따라서 피보나치 결과 반환
        d[x] = fibo(x - 1) + fibo(x - 2);
        return d[x];
    }
```

- 시간복잡도 : $O(N)$
  - $f(1)$부터 $f(N)$까지의 각 결과값은 생애 단 한 번만 계산되기 때문

- DP를 쓰면 위 그림처럼 함수 호출이 줄어든다
## 바텀업 방식 
> 작은 문제부터 차근차근 답을 도출하는 방식 (반복문 사용)
- 메모이제이션은 탑다운에 국한된 용어, 바텀업의 저장용 리스트는 **DP테이블**이라고 부름
```
// 첫 번째 피보나치 수와 두 번째 피보나치 수는 1
d = 1;
d = 1;
int n = 99; // 구하고자 하는 피보나치 수의 번호

// 피보나치 함수(Fibonacci Function) 반복문으로 구현 (보텀업 다이나믹 프로그래밍)
for (int i = 3; i <= n; i++) {
    d[i] = d[i - 1] + d[i - 2];
}
```

# 코테
- 메모이제이션은 사전자료형(java는 HashMap)을 사용할 수도 있다
  -  $a_0 \sim a_n$ 모든 인덱스의 값이 필요한 게 아니라, 드문드문 특정 인덱스의 값만 필요할 때 유용
- 피보나치 풀이법 기억하면 다른 DP에도 적용 가능 
- 3차원 리스트를 사용하는 복잡한 난이도도 나올 수 있음
  - 플로이드 워셜 알고리즘
- 완전 탐색으로 너무 오래걸리는 경우, 부분 문제들의 중복 여부를 확인해보고 DP사용
- **접근법**: 일단 부르트포스로 재귀함수 짯다가, 메모이제이션을 적용할 수 있으면 개선하는 방법으로 접근
- 가능하면 바텀업이 더 좋음. 재귀함수 오버헤드 때문에 (스택 넘칠수도 있음)

## A) 탑다운
```
import java.io.*;


public class Main{

    static int[] d;

    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        BufferedWriter bw = new BufferedWriter(new OutputStreamWriter(System.out));
      
        int X = Integer.parseInt(br.readLine());
        
        d = new int[X+1];
        int result = dp(X);
        bw.write(result + "");
        bw.flush();
        bw.close();


    }

    // 숫자input 그 숫자가 1되는 최소횟수output
    static int dp(int n) {
        if(n==1) return 0;

        if(d[n]!=0) return d[n];

        int min = dp(n-1) + 1;

        if(n%5==0) {
            min = Math.min(min, dp(n/5)+1);
        }
        if(n%3==0) {
            min = Math.min(min, dp(n/3)+1);
        }
        if(n%2==0) {
            min = Math.min(min, dp(n/2)+1);
        }

        return d[n] = min;

    }

}
```

- 시간복잡도 : $O(X)$
## A) 바텀업
```
import java.io.*;


public class Main{

    

    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        BufferedWriter bw = new BufferedWriter(new OutputStreamWriter(System.out));
      
        int X = Integer.parseInt(br.readLine());
        int[] d = new int[X+1];
        
        for(int i=2;i<=X;i++) {
            int min = d[i-1] + 1;

            if(i%5==0) {
                min = Math.min(min, d[i/5]+1);
            }
            if(i%3==0) {
                min = Math.min(min, d[i/3]+1);
            }
            if(i%2==0) {
                min = Math.min(min, d[i/2]+1);
            }

            d[i] = min;
        }

        bw.write(d[X] + "");
        bw.flush();
        bw.close();


    }



}
```

- 시간복잡도 : $O(X)$

## A) 바텀업
- d[i] : i번째까지의 최대치
  1) i번째 창고를 턴다 : d[i-2] + ki
  2) i번째 창고를 안턴다 : d[i-1]
```
import java.io.*;
import java.util.*;

public class Main{

    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        BufferedWriter bw = new BufferedWriter(new OutputStreamWriter(System.out));
      
        int N = Integer.parseInt(br.readLine());
        StringTokenizer st = new StringTokenizer(br.readLine());
        int[] storage = new int[N];
        for(int i=0;i<N;i++) {
            storage[i] = Integer.parseInt(st.nextToken());
        }
        int[] d = new int[N];
        d[0] = storage[0];
        d[1] = Math.max(storage[0],storage[1]);
        
        for(int i=2;i<N;i++) {
            d[i] = Math.max(d[i-1],d[i-2]+storage[i]);
        }

        bw.write(d[N-1] + "");
        bw.flush();
        bw.close();


    }

}
```
