# 당장 좋은 것만 선택하는 그리디
> 지금 당장 가장 좋아보이는 선택만 반복
- 현재 선택이 나중에 미칠 영향은 고려x
- 정렬(플로이드워셜, 다익스트라 등)과 다르게, 그리디는 사전에 알고리즘을 암기하지 않아도 직관적으로 풀 가능성이 있음
  - 다익스트라는 엄밀히 말하면 그리디다. 그리디면서 암기해야되는 알고리즘
- 이 문제가 '현재 상황에서 가장 좋은 것만을 선택해도 되는지'를 파악하는 능력을 요구
  - 문제에서 '가장 큰 순서대로', '가장 작은 순서대로'와 같은 암시적인 기준을 주는 경우가 많다
  - 대체로 **정렬 알고리즘**과 짝을 이루어 출제되는 경우가 많다 
## P) 거스름돈
<img width="679" height="434" alt="image" src="https://github.com/user-attachments/assets/74530cc7-b0e2-45e3-9a43-1a9756f84cec" />

- 키워드 : 동전의 '최소 개수'
### A)
- "가장 큰 화폐 단위부터" 거슬러 준다.
- 500원 → 100원 → 50원 → 10원 순서대로 줄 수 있는 만큼 최대한 주면, 전체 동전의 개수가 최소가 됩니다.
```
public class Main {
    public static void main(String[] args) {
        int n = 1260;
        int count = 0;

        int[] coins = {500, 100, 50, 10};

        for (int coin : coins) {
            count += n / coin; // 해당 동전 최대 개수
            n %= coin;         // 남은 돈
        }

        System.out.println(count); // 6
    }
}
```
### 시간복잡도
- O(K)
- 거슬러 줄 돈(N)과 상관 없이, 동전의 개수(K)에만 영향을 받음 (매우 효율적) 
## 그리디 알고리즘의 정당성
> 이 문제에 그리디를 써도 되는가?
- 거스름돈 문제 
  - 동전들 중 **큰 단위가 항상 작은 단위의 배수**이기 때문에, 작은 동전들로 새로운 조합이 나올 수 없음
  - 만약 배수가 아니라면? (500원, 400원, 100원)
    - **목표:** 800원 거슬러 주기
    - **그리디의 선택:** 500원(1개) + 100원(3개) = 4개
    - **제 최적의 해:** 400원(2개) = 2개
- 그리디는 항상 이렇게 정당성을 검토할 수 있어야 한다
- 문제 유형 파악이 어렵다면
  - 우선 그리디 고려
  - 해법 못찾으면 그뒤에 DP나 그래프 알고리즘 고려 
- 밑에 나오는 문제들은 모두 난이도 '하'이고, 30분이내에 풀어야 기업코테가 할만한다 
# P) 큰 수의 법칙
<img width="699" height="837" alt="image" src="https://github.com/user-attachments/assets/63061b37-1826-41d7-9b7d-8608cf71797c" />

- N: 배열의 크기 (숫자의 개수)
- M: 숫자가 더해지는 총 횟수
- K: 특정 인덱스의 숫자가 연속해서 더해질 수 있는 최대 횟수\
## A1
```
public class Main {
    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        StringTokenizer st = new StringTokenizer(br.readLine());

        int N = Integer.parseInt(st.nextToken());
        int M = Integer.parseInt(st.nextToken());
        int K = Integer.parseInt(st.nextToken());

        int[] arr = new int[N];
        st = new StringTokenizer(br.readLine());
        for(int i = 0; i < N; i++) {
            arr[i] = Integer.parseInt(st.nextToken());
        }

        Arrays.sort(arr);
        int first = arr[N - 1];
        int second = arr[N - 2];

        // --- 수학적 공식 적용 ---
        // 가장 큰 수가 더해지는 횟수 계산
        long count = (long)(M / (K + 1)) * K;
        count += M % (K + 1);

        long sum = 0;
        sum += count * first;           // 가장 큰 수 더하기
        sum += (M - count) * second;    // 두 번째로 큰 수 더하기
        // -----------------------

        System.out.println(sum);
        br.close();
    }
}
```

- 시간복잡도 : O(1)
- 수학적 공식 활용
