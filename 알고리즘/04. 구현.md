<img width="538" height="240" alt="image" src="https://github.com/user-attachments/assets/a5863ecd-b131-4bb5-a3c7-9065b2549d4c" /># 아이디어를 코드로 바꾸는 구현
- **구현 :** 머릿속에 있는 알고리즘을 소스코드로 바꾸는 과정
- "풀이는 떠오르는데 코드로 옮기기 어렵다"라고 느끼는 문제들이 이 유형
## 피지컬로 승부하기
- 책에서는 구현 능력을 **피지컬**에 비유하고 있습니다.
  - **개념:** 머릿속에 있는 알고리즘을 소스코드로 정확하고 빠르게 바꾸는 과정입니다.
  - **비유:** 스타크래프트에서 전략(알고리즘)을 잘 짜도 마우스 컨트롤(구현)이 안 되면 지는 것과 같습니다.
  - **특징:** 이론적으로는 쉽지만, 막상 코드로 옮기려면 조건이 까다롭거나 코드가 길어져서 실수하기 쉬운 유형입니다.
- 초보자가 특히 어려워하는 '구현' 유형은 보통 다음과 같은 특징을 가집니다.
  - **지나치게 긴 코드:** 알고리즘은 단순한데 처리해야 할 조건문이 너무 많을 때.
  - **까다로운 출력 포맷:** 특정 소수점 자리까지 출력하거나, 특수 문자를 포함해야 할 때.
  - **문자열 파싱:** 입력받은 문자열을 특정 단위로 끊어서 리스트에 넣는 등의 작업.
  - **방대한 라이브러리 지식:** 예를 들어 '순열'을 구할 때 직접 구현할 수도 있지만, 파이썬의 itertools 같은 표준 라이브러리를 모르면 시간이 훨씬 오래 걸립니다.
- 이 책에서는, 2가지를 구현 유형으로 다루고 있다
  - **완전 탐색 (Brute Force) :** 모든 경우의 수를 주저 없이 다 계산하는 방식
  - **시뮬레이션 (Simulation) :** 문제에서 제시한 알고리즘을 한 단계씩 차례대로 수행
- 구현 문제는 코딩 테스트 환경 제약을 알아야 하므로 아래서 다룬다 
## 구현 시 고려해야 할 메모리 제약 사항
### 변수의 표현 범위
- Java에서 int는 약 21억까지 담을 수 있다 (4바이트)
- 값이 커진다 싶으면 그냥 long을 쓰자 (8바이트)
### 배열의 크기
- 문제에서 보통 128MB같은 조건이 붙는다
- int 배열 크기 1,000만이 약 40MB 정도이다. 
## 채점 환경
- 보통
  - 시간 제한 : 1초
  - 메모리 제한 : 128MB
## 구현 문제에 접근하는 방법
- 문제의 길이가 길고 사소한 입력 조건이 많습니다.
- 고차원적인 사고력보다는 문법에 익숙함이 중요하며, 지레 겁먹지 않고 문제 요구사항을 그대로 코드로 옮기는 능력이 필요합니다.
- 문자열 처리, 큰 정수 처리 등이 자주 출제됩니다. (따라서 파이썬이 유리)

### A)
- 시뮬레이션 유형 : 일련의 명령에 따라서 개체를 차례대로 이동
- 코테 난이도낮은 1~2번 문제는 그리디나 구현인 경우가 많다. 가장 기본 난이도라서.
```
// N+1 정사각형 배열 만들기
// 방향벡터 만들기
// for문 돌려서 입력된 문자열이 몇번째 인덱스인지 파악
// 그 인덱스의 방향벡터 방향으로 이동
// 경계선넘으면 continue
public class Main{

    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        BufferedWriter bw = new BufferedWriter(new OutputStreamWriter(System.out));
      
        int N = Integer.parseInt(br.readLine());
        int c = 1;
        int r = 1;

        StringTokenizer st = new StringTokenizer(br.readLine());
        int[] dc = {-1,1,0,0};
        int[] dr = {0,0,-1,1};
        char[] ch = {'L','R','U','D'};
        while(st.hasMoreTokens()) { // 발상
            char nextMove = st.nextToken().charAt(0);

            for(int i=0;i<4;i++) {
                if(ch[i]==nextMove) {
                    int nc = c + dc[i];
                    int nr = r + dr[i];

                    if(nc<1 || nc>N || nr<1 || nr>N) break;

                    c = nc;
                    r = nr;
                } 
            }
        }

        bw.write(r+" "+c);
        bw.flush();
        bw.close();


    }
}
```

- 시간복잡도 : $O(M)$
- M : 여기서 $M$은 이동 횟수, 즉 StringTokenizer로 읽어오는 명령어의 개수

```
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        BufferedWriter bw = new BufferedWriter(new OutputStreamWriter(System.out));
      
        int N = Integer.parseInt(br.readLine());
        int cnt = 0;

        for(int i=0;i<=N;i++) {
            for(int j=0;j<60;j++) {
                for(int k=0;k<60;k++) {
                    // i, j, k중 3이 있으면 카운트
                    if(i%10==3 || j/10 ==3 || j%10 == 3 ||k/10 ==3 || k%10 == 3 ) cnt++;
                }
            }
        }

        bw.write(cnt+" ");
        bw.flush();
        bw.close();


    }
}
```

- 시간복잡도 : $O(N)$
- 60 x 60 x N

```
public class Main{

    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        BufferedWriter bw = new BufferedWriter(new OutputStreamWriter(System.out));
      
        String str = br.readLine();

        int row = str.charAt(1) - '0';
        int col = str.charAt(0) - 'a' + 1;
        int cnt = 0;

        int[] dr = {2,2,-2,-2,1,-1,1,-1};
        int[] dc = {1,-1,1,-1,2,2,-2,-2};

        for(int i=0;i<8;i++) {
            int nRow = row + dr[i];
            int nCol = col + dc[i];
            if(0<nRow && nRow<9 && 0<nCol && nCol<9) cnt++;
        }



        bw.write(cnt + "");
        bw.flush();
        bw.close();


    }
}
```

- 유형 : 시뮬레이션, 구현
- 방향벡터 사용
- 1a같은 입력도 가능한 문제 : `Character.isDigit(str.charAt(0))` 활용해서 방어적 프로그래밍
- 시간복잡도 :  $O(1)$
  - 8번 고정
