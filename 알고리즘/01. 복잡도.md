# 복잡도(Complexity)
> 알고리즘이 얼마나 효율적인지 나타내는 척도
- **시간 복잡도** : 입력 크기에 따른 연산 횟수
- **공간 복잡도** : 입력 크기에 따른 메모리 차지 용량
- 동일한 기능을 수행한다면, 일반적으로 복잡도가 낮은게 좋은 알고리즘
- 시간 복잡도와 공간 복잡도는 보통 trade-off이다
  - ex) 메모제이션 : 메모리를 더 사용해서 시간을 단축 
# 시간 복잡도
> 입력 크기에 따른 연산 횟수 
- 알고리즘 문제를 풀 때, 복잡도라고 하면 보통 시간 복잡도이다
- 시간 복잡도는 **실제 시간**이 아니라 **연산 횟수**가 기준이다
  - 실제 실행 시간은 컴퓨터마다 다르기 때문이다
- **입력크기** : 배열의 크기, 정점과 간선의 수 등등
- **단위연산** : 비교, 할당 등 시간복잡도에 가장 영향이 큰 연산
- **시간복잡도** : **입력크기**에 따라 **단위연산**이 몇번 수행되는가
## Big-O 표기법
> 가장 빠르게 증가하는 항만 고려하는 표기법 
- N이 커질수록 나머지 항은 무시 가능해짐
## 대표 시간 복잡도
| 복잡도      | 의미    |
| -------- | ----- |
| O(1)     | 상수 시간 |
| O(logN)  | 로그 시간 |
| O(N)     | 선형 시간 |
| O(NlogN) | 로그 선형 |
| O(N²)    | 이차 시간 |
| O(N³)    | 삼차 시간 |
| O(2ⁿ)    | 지수 시간 |
## 코딩 테스트 
- N이 작을땐 상수항도 무시할 수 없다
- 보통 1초 ≈ 10⁸ 연산으로 어림잡는다
- O(N^3)을 넘어가면 보통 사용하기 어렵다
