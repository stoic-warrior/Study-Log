# 꼭 필요한 자료구조 기초
- **탐색** : 많은 양의 데이터 중에서 원하는 데이터를 찾는 과정
  - 그래프, 트리 등에서의 탐색 문제를 자주 다룸
  - 대표적인 탐색 알고리즘이 DFS,BFS
  - DFS, BFS를 제대로 이해하려면 스택과 큐같은 자료구조에 대한 이해가 있어야 함 
- **자료구조** : 데이터를 표현하고 관리하고 처리하기 위한 구조
- **스택과 큐의 핵심함수**
  - **삽입(Push)**: 데이터를 넣는 작업
  - **삭제(Pop)**: 데이터를 꺼내는 작업
- **주의**
  - **오버플로(Overflow)**: 이미 꽉 찬 자료구조에 데이터를 더 넣으려고 할 때 발생합니다. (넘침)
  - **언더플로(Underflow)**: 데이터가 하나도 없는데 삭제 연산을 수행하려고 할 때 발생합니다. (비어있음)
## 스택
> 입구와 출구가 하나인 원통
<img width="132" height="188" alt="image" src="https://github.com/user-attachments/assets/ceb26408-e3cd-45ad-945e-831ab500b79a" />

- **LIFO (Last-In First-Out)**: 나중에 들어온 데이터가 가장 먼저 나가는 구조입니다.
- **FILO (First-In Last-Out)**: 반대로 먼저 들어온 데이터는 가장 나중에 나가게 됩니다.
- 스택은 **뒤로 가기, 실행 취소(Ctrl+Z), 함수 호출 기록** 등에 사용
### 예시
- <img width="509" height="682" alt="image" src="https://github.com/user-attachments/assets/f5ef54a1-f41c-4713-a110-4e8bcfd05ac6" />
- <img width="467" height="351" alt="image" src="https://github.com/user-attachments/assets/87004c8b-c1b9-4320-acaa-7b347380c6ba" />

```
import java.util.ArrayDeque;
import java.util.Deque;

public class Main {
    public static void main(String[] args) {
        // Deque 인터페이스를 통해 ArrayDeque 생성
        Deque<Integer> stack = new ArrayDeque<>();

        // 1. 삽입 (Push)
        stack.push(5);
        stack.push(2);
        stack.push(3);
        stack.push(7);

        // 2. 최상단 제거 (Pop)
        stack.pop(); // 7이 나감

        stack.push(1);
        stack.push(4);

        // 3. 최상단 제거 (Pop)
        stack.pop(); // 4가 나감

        // 결과 확인
        System.out.println(stack); // (상단부터 출력됨)
        
        // 4. 맨 위의 값만 확인 (Peek)
        System.out.println(stack.peek()); // 1
    }
}
```
- **Push**: `addLast()` 또는 `push()`
- **Pop**: `removeLast()` 또는 `pop()`
- Java 공식 문서에서는 `Stack` 클래스보다 `ArrayDeque`를 사용하는 것을 더 권장 (stack은 옛날에 설계되어 성능과 안정성 둘다 낮음)
- `Deque<Integer>`처럼 스택에 들어갈 데이터의 타입을 미리 정해야 함 (제네릭)

```
import java.util.ArrayDeque;
import java.util.Queue;

public class Main {
    public static void main(String[] args) {
        // Queue 인터페이스로 ArrayDeque 생성
        Queue<Integer> q = new ArrayDeque<>();

        q.offer(5); // 삽입
        q.offer(2);
        q.poll();   // 삭제 (5가 나감)
        
        System.out.println(q.peek()); // 맨 앞의 값 확인 (2)
    }
}
```
## 재귀 함수
> 자기 자신을 다시 호출하는 함수
```
public class RecursiveExample {
    // 재귀 함수 정의
    public static void recursiveFunction() {
        System.out.println("재귀 함수를 호출합니다.");
        recursiveFunction(); // 자기 자신을 다시 호출
    }

    public static void main(String[] args) {
        // 재귀 함수 실행
        recursiveFunction();
    }
}
```

- 컴퓨터 내부에서 재귀함수는 스택 자료구조를 이용 (메모리의 스택공간) 
- 종료조건이 없으면, StackOverflowError가 발생하며 프로그램이 강제로 종료
### 반드시 필요한 '종료 조건'
```
public class RecursiveExitExample {
    public static void recursiveFunction(int i) {
        // 100번째 호출되었을 때 종료되도록 조건 명시 (Base Case)
        if (i == 100) {
            return;
        }

        System.out.println(i + "번째 재귀 함수에서 " + (i + 1) + "번째 재귀 함수를 호출합니다.");
        
        recursiveFunction(i + 1);
        
        System.out.println(i + "번째 재귀 함수를 종료합니다.");
    }

    public static void main(String[] args) {
        // 1부터 시작하여 재귀 함수 호출
        recursiveFunction(1);
    }
}
```

- 위의 if문 처럼, 재귀함수는 종료조건이 반드시 필요 (무한루프 방지)
### 팩토리얼 예제
```
public class FactorialExample {

    // 1. 반복적으로 구현한 n!
    public static int factorialIterative(int n) {
        int result = 1;
        // 1부터 n까지의 수를 차례대로 곱하기
        for (int i = 1; i <= n; i++) {
            result *= i;
        }
        return result;
    }

    // 2. 재귀적으로 구현한 n!
    public static int factorialRecursive(int n) {
        // n이 1 이하인 경우 1을 반환 (종료 조건)
        if (n <= 1) {
            return 1;
        }
        // n! = n * (n - 1)! 공식 그대로 작성
        return n * factorialRecursive(n - 1);
    }

    public static void main(String[] args) {
        // 각각의 방식으로 구현한 5! 출력
        System.out.println("반복적으로 구현: " + factorialIterative(5));
        System.out.println("재귀적으로 구현: " + factorialRecursive(5));
    }
}
```

- 위 반복문과 재귀 함수의 결과는 같다
- 재귀 함수의 장점은 코드가 간결해진다는 것이다 (위 예시에선 아니지만..)
- 재귀 함수는 수학의 **점화식**을 활용한다 (점화식은 DP에서도 쓰인다)

# 탐색 알고리즘 DFS/BFS
## 그래프를 표현하는 두 가지 방법
- 코테는 인접 리스트를 많이 씀 (공간복잡도, 시간복잡도) 
### 인접 행렬 (Adjacency Matrix)
> 2차원 배열을 사용
<img width="547" height="166" alt="image" src="https://github.com/user-attachments/assets/1bb63a73-abdf-40fc-9996-b339676e39c1" />

```
int INF = 1000000000;

int[][] graph = {
    {0, 7, 5},
    {7, 0, INF},
    {5, INF, 0}
};
```
- 당장 두 노드의 연결여부를 확인할 때 유리 (플로이드 워셜?)
- 나머지는 인접리스트보다 DFS에 불리함
### 인접 리스트 (Adjacency List)
> 연결 리스트(Java에서는 ArrayList 등)를 사용
<img width="347" height="241" alt="image" src="https://github.com/user-attachments/assets/93ee7816-9115-4146-a2b1-ff3118c29a7e" />

```
import java.util.*;

ArrayList<int[]>[] graph = new ArrayList[3];
for (int i = 0; i < 3; i++) graph[i] = new ArrayList<>();

graph[0].add(new int[]{1, 7});
graph[0].add(new int[]{2, 5});

graph[1].add(new int[]{0, 7});

```
## DFS
> 스택or 재귀 함수로 구현
- 1️⃣ 시작 노드를 스택에 넣는다.
- 2️⃣ 스택이 빌 때까지 반복한다.
  - 1️⃣ 스택에서 노드를 하나 꺼낸다. (pop)
    - 1 이미 방문한 노드라면 무시한다.
    - 2 방문하지 않은 노드라면:
      - 방문 처리한다.
      - 해당 노드를 출력/처리한다.
      - 인접 노드 중 방문하지 않은 노드를 스택에 넣는다.
```
        // 1️⃣ 시작 노드를 스택에 넣는다.
        stack.push(1);

        // 2️⃣ 스택이 빌 때까지 반복
        while (!stack.isEmpty()) {

            // 1️⃣ 스택에서 노드 꺼내기
            int now = stack.pop();

            // 1️⃣-1 이미 방문했으면 무시
            if (visited[now]) continue;

            // 1️⃣-2 방문하지 않았다면
            visited[now] = true;              // 방문 처리
            System.out.print(now + " ");      // 출력

            // 인접 노드 push
            for (int next : graph[now]) {
                if (!visited[next]) {
                    stack.push(next);
                }
            }
        }
```
## BFS
> 큐로 구현
- 시작 노드를 큐에 넣고 방문 처리한다.
- 큐가 빌 때까지 반복한다.
  - 큐에서 노드를 하나 꺼낸다. (poll)
  - 해당 노드를 출력/처리한다.
  - 인접 노드 중 방문하지 않은 노드를:
    - 큐에 넣는다.
    - 방문 처리한다.
```
        // 1️⃣ 시작 노드를 큐에 넣고 방문 처리
        queue.offer(1);
        visited[1] = true;

        // 2️⃣ 큐가 빌 때까지 반복
        while (!queue.isEmpty()) {

            // 2-1️⃣ 큐에서 노드 꺼내기
            int now = queue.poll();

            // 2-2️⃣ 출력
            System.out.print(now + " ");

            // 2-3️⃣ 인접 노드 처리
            for (int next : graph[now]) {
                if (!visited[next]) {
                    queue.offer(next);     // 큐에 넣기
                    visited[next] = true;  // 방문 처리 (여기서!)
                }
            }
        }
    }
```

- 큐에 넣을때 방문처리 하는 이유는, BFS 구조상 그렇게 해야 중복push를 막을 수 있기 때문이다
