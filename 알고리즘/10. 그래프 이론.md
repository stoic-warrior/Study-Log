# 이미 배운 내용을 훑어보자
- BFS/DFS, 최단경로에서 배운 것들은 그래프 알고리즘의 한 유형
- 나머지 그래프 알고리즘을 여기서 다룬다.
- 그전에 전에꺼 훑어보자 
## 그래프(Graph)와 트리(Tree)의 비교
- 그래프는 노드(Node)와 노드 사이를 잇는 간선(Edge)으로 이루어진 자료구조
- 트리는 그래프의 한 종류이지만 명확한 차이점이 있다

| 구분 | 그래프 (Graph) | 트리 (Tree) |
| :--- | :--- | :--- |
| **방향성** | 방향(Directed) 또는 무방향(Undirected) | 방향 그래프 (컴퓨터공학 기준) |
| **순환성** | 순환(Cycle) 및 비순환(Acyclic) 모두 가능 | **비순환(Acyclic)**만 가능 |
| **루트 노드** | 없음 (모든 노드가 대등함) | **존재함** (하나의 뿌리 노드) |
| **관계** | 부모-자식 관계 없음 | **부모-자식 관계 존재** (계층적) |
| **모델 종류** | 네트워크 모델 (Network Model) | 계층 모델 (Hierarchical Model) |

## 그래프를 구현하는 두 가지 방식
- ① 인접 행렬 (Adjacency Matrix)
  - 방식: 2차원 배열을 사용하여 연결 관계를 표현합니다.
  - 장점: 두 노드의 연결 여부나 간선 비용을 $O(1)$ 시간으로 즉시 확인 가능합니다.
  - 단점: 노드가 $V$개일 때 $O(V^2)$의 메모리 공간이 필요하여 메모리 효율이 낮습니다.
  - 활용: 플로이드 워셜 알고리즘 등.
- ② 인접 리스트 (Adjacency List)
  - 방식: 리스트(혹은 연결 리스트)를 사용하여 연결된 정보만 저장합니다.
  - 장점: 간선의 개수 $E$만큼인 $O(E)$의 메모리만 사용하여 공간 효율적입니다.
  - 단점: 특정 노드 간 연결을 확인하는 데 $O(V)$의 시간이 소요될 수 있습니다.
  - 활용: 다익스트라 최단 경로 알고리즘 등.
```
// 인접 행렬
int[][] dist = new int[N+1][N+1];

// 인접 리스트
ArrayList<Node>[] adj = new ArrayList<>();
```
## 팁
- 문제 식별: "여러 개체가 서로 연결되어 있다"는 표현이 나오면 그래프 알고리즘을 떠올려야 합니다.
- 알고리즘 선택 기준:
  - 노드 개수가 적다면 플로이드 워셜(인접 행렬)이 구현하기 편합니다.
  - 노드와 간선 개수가 모두 많다면 다익스트라(인접 리스트 + 우선순위 큐)를 사용하는 것이 유리합니다.
- 연관 개념: 크루스칼 알고리즘은 그리디, 위상 정렬은 큐/스택 자료구조를 활용하여 구현됩니다.
# 서로소 집합
- 서로소 집합 : 공통원소가 없는 두 집합
- 서로소 집합 자료구조 : 서로소 부분 집합들로 나누어진 원소들의 데이터를 처리하기 위한 자료구조
- 유니온-파인드 자료구조라고도 부른다
- 연산
  - union : 두 원소가 속한 집합 합치기, 보통 더 작은 원소가 부모가 되도록 구현
  - find : 어떤 집합에 속했는지 확인 (루트노드 확인)
```
import java.io.*;
import java.util.*;

public class Main {
    static int[] parent;

    // 초기화: 처음에는 자기 자신이 대표자
    static void makeSet(int n) {
        parent = new int[n + 1];

        for (int i = 1; i <= n; i++) {
            parent[i] = i;
        }
    }

    // find: x가 속한 집합의 대표자 찾기 + 경로 압축
    static int find(int x) {
        if (parent[x] == x) {
            return x;
        }

        // 경로 압축 핵심
        parent[x] = find(parent[x]);

        return parent[x];
    }

    // union: 두 집합 합치기
    static void union(int a, int b) {
        int rootA = find(a);
        int rootB = find(b);

        if (rootA == rootB) {
            return;
        }

        // 한쪽 대표자를 다른 쪽 대표자 밑으로 붙임
        parent[rootB] = rootA;
    }

    // 같은 집합인지 확인
    static boolean isSameSet(int a, int b) {
        return find(a) == find(b);
    }
}
```

## 경로 압축 (Path Compression)
- (P) 기본적인 Find 함수는 트리가 한쪽으로 치우칠 경우(예: $1 \leftarrow 2 \leftarrow 3 \leftarrow 4$) 시간 복잡도가 $O(V)$로 늘어날 수 있습니다.
- (S) 경로 압축 기법
- Find 함수를 호출할 때 방문하는 노드들이 직접 루트 노드를 가리키도록 업데이트
```
static int find(int x) {
    if (parent[x] == x) {
        return x;
    }

    parent[x] = find(parent[x]); //   핵심
    return parent[x];
}
```
```
// 더 짧은버전
static int find(int x) {
    if (parent[x] != x) {
        parent[x] = find(parent[x]);
    }
    return parent[x];
}
```

## 사이클 판별 (Cycle Detection)
- 그래프의 모든 간선에 대해 아래를 수행
  - 간선 두 노드의 루트 노드를 확인
  - 다른 루트면 : 두 노드 Union
  - 같은 루트면 : 사이클이 확인된 것
```
import java.io.*;
import java.util.*;

public class Main {
    static int[] parent;

    static int find(int x) {
        if (parent[x] == x) return x;
        return parent[x] = find(parent[x]); // 경로 압축
    }

    static boolean union(int a, int b) {
        int rootA = find(a);
        int rootB = find(b);

        // 이미 같은 집합이면 사이클 발생
        if (rootA == rootB) {
            return true;
        }

        // 합치기
        parent[rootB] = rootA;
        return false;
    }

    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));

        StringTokenizer st = new StringTokenizer(br.readLine());
        int v = Integer.parseInt(st.nextToken()); // 정점 개수
        int e = Integer.parseInt(st.nextToken()); // 간선 개수

        parent = new int[v + 1];

        for (int i = 1; i <= v; i++) {
            parent[i] = i;
        }

        boolean hasCycle = false;

        for (int i = 0; i < e; i++) {
            st = new StringTokenizer(br.readLine());
            int a = Integer.parseInt(st.nextToken());
            int b = Integer.parseInt(st.nextToken());

            if (union(a, b)) {
                hasCycle = true;
                break;
            }
        }

        if (hasCycle) {
            System.out.println("사이클 발생");
        } else {
            System.out.println("사이클 없음");
        }
    }
}
```
