# 이미 배운 내용을 훑어보자
- BFS/DFS, 최단경로에서 배운 것들은 그래프 알고리즘의 한 유형
- 나머지 그래프 알고리즘을 여기서 다룬다.
- 그전에 전에꺼 훑어보자 
## 그래프(Graph)와 트리(Tree)의 비교
- 그래프는 노드(Node)와 노드 사이를 잇는 간선(Edge)으로 이루어진 자료구조
- 트리는 그래프의 한 종류이지만 명확한 차이점이 있다

| 구분 | 그래프 (Graph) | 트리 (Tree) |
| :--- | :--- | :--- |
| **방향성** | 방향(Directed) 또는 무방향(Undirected) | 방향 그래프 (컴퓨터공학 기준) |
| **순환성** | 순환(Cycle) 및 비순환(Acyclic) 모두 가능 | **비순환(Acyclic)**만 가능 |
| **루트 노드** | 없음 (모든 노드가 대등함) | **존재함** (하나의 뿌리 노드) |
| **관계** | 부모-자식 관계 없음 | **부모-자식 관계 존재** (계층적) |
| **모델 종류** | 네트워크 모델 (Network Model) | 계층 모델 (Hierarchical Model) |

## 그래프를 구현하는 두 가지 방식
- ① 인접 행렬 (Adjacency Matrix)
  - 방식: 2차원 배열을 사용하여 연결 관계를 표현합니다.
  - 장점: 두 노드의 연결 여부나 간선 비용을 $O(1)$ 시간으로 즉시 확인 가능합니다.
  - 단점: 노드가 $V$개일 때 $O(V^2)$의 메모리 공간이 필요하여 메모리 효율이 낮습니다.
  - 활용: 플로이드 워셜 알고리즘 등.
- ② 인접 리스트 (Adjacency List)
  - 방식: 리스트(혹은 연결 리스트)를 사용하여 연결된 정보만 저장합니다.
  - 장점: 간선의 개수 $E$만큼인 $O(E)$의 메모리만 사용하여 공간 효율적입니다.
  - 단점: 특정 노드 간 연결을 확인하는 데 $O(V)$의 시간이 소요될 수 있습니다.
  - 활용: 다익스트라 최단 경로 알고리즘 등.
## 팁
- 문제 식별: "여러 개체가 서로 연결되어 있다"는 표현이 나오면 그래프 알고리즘을 떠올려야 합니다.
- 알고리즘 선택 기준:
  - 노드 개수가 적다면 플로이드 워셜(인접 행렬)이 구현하기 편합니다.
  - 노드와 간선 개수가 모두 많다면 다익스트라(인접 리스트 + 우선순위 큐)를 사용하는 것이 유리합니다.
- 연관 개념: 크루스칼 알고리즘은 그리디, 위상 정렬은 큐/스택 자료구조를 활용하여 구현됩니다.
