# 최신 출제 경향과 준비 방향
- 대기업 공채처럼 지원자가 많은 상황에서 효과적으로 지원자를 선별하기 위한 장치
- 보통 2~5시간 동안 여러 개의 알고리즘 문제
- 정확한 알고리즘을 사용하여 얼마나 빠르고 정확하게 문제를 해결했는지를 토대로 순위를 매김

- 지나치게 어려운 알고리즘 지식은 불필요 (그런건 대회에서나 씀) 
- 대부분의 문제는 기초적인 알고리즘 안에서 출제 (그리디, 구현, BFS/DFS 등) 
- 어려운 문제는 기초 알고리즘 여러개를 복합적으로 섞어서 푸는 문제 
## 기업별 문제 출제 경향
- 보통 2~5시간 동안 8개 이하의 문제를 풀어야 함
- 시간이 촉박하여 심리적 부담으로 평소보다 기량이 안나 올 수 있으니 연습해 두어야 함
- 몇 문제 맞춰야 합격할까?
  - 대회나 코테에서 올솔(문제 다푸는)하는 사람은 상위5%밖에 안됨
  - 설문조사 결과, 빅테크 합격자는 평균 69%의 문제를 풀엇다 응담, 불합격자는 평균 38%의 문제를 풀었다 응답
    - 이것이 진실이라면, 절반이상 안정적으로 푸는게 중요해 보인다
  - 체감난이도는 10점만점에 6점정도라고 한다 
### 카카오
- 알고리즘 대회 수준(X)
- 현업 상황 기반 문제
- 특징
  - 문자열을 처리해야 하는 구현 문제 자주 출제 
  - 다양한 케이스를 고려해야 하는 문제
### 삼성
- 문제를 정확히 읽고 예외 상황을 적절히 처리하는 소스코드 작성를 작성하는 유형이 자주 출제 
- 완전 탐색(Brute Force), DFS/BFS, 구현 유형을 가장 선호
- 완전 탐색 :  모든 경우의 수를 따져야 함, 체력 싸움
