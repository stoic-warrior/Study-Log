# 기준에 따라 데이터를 정렬
## 정렬 알고리즘 개요
> 데이터를 특정한 기준에 따라 순서대로 나열
- 프로그램에서 데이터를 가공할 때, 오름차순 내림차순 등 어떤 식으로든 정렬해서 사용하는 경우가 많다
- 데이터를 정렬하면 이진 탐색도 가능해진다
- 알고리즘의 효율성을 쉽게 이해할 수 있게 만든다
  - 상황에 부적절한 정렬 알고리즘은 프로그램을 비효율적으로 만듬
- 여기서는 오름차순을 기준으로 설명한다. 
## 선택 정렬
> 가장 작은 것을 **선택**해서 맨 앞으로 보낸다
- 가장 원시적인 방법

```
for (int i = 0; i < array.length; i++) {
    int minIndex = i; // 가장 작은 원소의 인덱스
    for (int j = i + 1; j < array.length; j++) {
        if (array[minIndex] > array[j]) {
            minIndex = j;
        }
    }
    
    // 스와프(Swap): 두 원소의 위치 변경
    int temp = array[i];
    array[i] = array[minIndex];
    array[minIndex] = temp;
}

// 결과 출력
for (int val : array) {
    System.out.print(val + " ");
}
```

- 스와프 : 두 변수의 위치를 바꾸는 것 (temp 활용)
- 시간복잡도
  - 연산횟수 : $N + (N-1) + (N-2) + ... + 2$
  - = **$\frac{1}{2}N^2 - \frac{1}{2}N$**
  - $O(N^2)$
- 효율은 매우 구리지만, 특정 배열에서 가장 작은 데이터를 찾는 일은 코테에서 잦으므로 이 소스코드에 익숙해져야 한다
## 삽입 정렬
> 각 데이터를 적절한 위치에 삽입
- 두 번째 데이터부터 시작합니다 (첫 번째는 이미 정렬되어 있다고 가정)
- 선택된 데이터를 왼쪽의 정렬된 데이터들과 비교하며, 자신보다 작은 데이터를 만날 때까지 왼쪽으로 이동
- 자신보다 작은 데이터를 만나면 그 바로 오른쪽에 멈춤

```
// 두 번째 원소부터 시작
        for (int i = 1; i < array.length; i++) {
            // 인덱스 i부터 1까지 감소하며 왼쪽 원소와 비교
            for (int j = i; j > 0; j--) {
                // 왼쪽 원소가 더 크다면 스와프(Swap)
                if (array[j] < array[j - 1]) {
                    int temp = array[j];
                    array[j] = array[j - 1];
                    array[j - 1] = temp;
                } 
                // 자신보다 작은 데이터를 만나면 그 위치에서 멈춤
                else {
                    break;
                }
            }
        }
```

- 데이터가 거의 정렬되어 있을 때 압도적으로 빠르다
- 평균 시간 복잡도: $O(N^2)$ (선택 정렬과 비슷함)
- 최선의 경우 (Best Case): 이미 정렬되어 있다면 한 번씩만 비교하고 끝나므로 **$O(N)$**
## 퀵 정렬
> 기준(피벗)을 정하고 그보다 큰 값과 작은 값을 나누는 과정을 반복
- 퀵정렬은 여기에 나온것중에 가장 많이 쓰이는 정렬 알고리즘
- 퀵정렬과 병합정렬은 대부분의 언어 정렬 라이브러리의 근간이 된다
- **피벗(Pivot)** : 기준 데이터
- 퀵정렬 종류에 따라 피벗을 정하는 방법이 다르다. 여기서는 호어 분할을 쓰자
- **호어 분할** : 리스트의 첫 번째 데이터를 피벗으로 정한다
- 1피벗을 설정한 후, 왼쪽에서는 피벗보다 큰 데이터를 찾고, 오른쪽에서는 피벗보다 작은 데이터를 찾아 서로 위치를 교환
- 2엇갈리면 작은데이터와 피벗의 위치를 교환
- 3피벗의 왼쪽과 오른쪽으로 분할 (왼쪽은 피벗보다 작은 숫자들, 오른쪽은 피벗보다 큰 숫자들이 모이게 된다)
- 분할된 각각의 리스트에 1~3번 적용
- 재귀의 형태를 띠므로, 종료조건이 필요
  - 종료조건 : 리스트의 크기가 1

```
public class QuickSort {
    public static void main(String[] args) {
        int[] array = {5, 7, 9, 0, 3, 1, 6, 2, 4, 8};

        quickSort(array, 0, array.length - 1);

        // 결과 출력
        for (int i : array) {
            System.out.print(i + " ");
        }
    }

    public static void quickSort(int[] array, int start, int end) {
        if (start >= end) return; // 원소가 1개인 경우 종료

        int pivot = start; // 피벗은 첫 번째 원소
        int left = start + 1;
        int right = end;

        while (left <= right) {
            // 피벗보다 큰 데이터를 찾을 때까지 반복
            while (left <= end && array[left] <= array[pivot]) {
                left++;
            }
            // 피벗보다 작은 데이터를 찾을 때까지 반복
            while (right > start && array[right] >= array[pivot]) {
                right--;
            }

            if (left > right) { // 엇갈렸다면 작은 데이터(right)와 피벗을 교체
                int temp = array[right];
                array[right] = array[pivot];
                array[pivot] = temp;
            } else { // 엇갈리지 않았다면 큰 데이터와 작은 데이터를 교체
                int temp = array[left];
                array[left] = array[right];
                array[right] = temp;
            }
        }

        // 분할 이후 왼쪽 부분과 오른쪽 부분에서 각각 정렬 수행
        quickSort(array, start, right - 1);
        quickSort(array, right + 1, end);
    }
}
```

| 상황 | 시간 복잡도 | 비고 |
| :--- | :---: | :--- |
| **평균 (Average Case)** | $O(N \log N)$ | 데이터가 무작위로 입력되었을 때 |
| **최악 (Worst Case)** | $O(N^2)$ | 데이터가 이미 정렬되어 있는 경우 (피벗이 한쪽으로 치우침) |

### 2. 왜 평균 $O(N \log N)$인가요?
- **높이 ($log N$):** 데이터의 개수가 $N$개일 때, 피벗이 매번 리스트를 절반에 가깝게 나눈다면 리스트의 높이는 약 $\log_2 N$이 됩니다.
- **너비 ($N$):** 각 층에서 데이터를 비교하고 분할하는 데 드는 연산은 약 $N$번입니다.
- **결과:** 따라서 전체적인 시간 복잡도는 $N \times \log N = O(N \log N)$이 됩니다.
## 계수 정렬
> 각 데이터가 몇 번 등장했는지 횟수를 세는 방식
- 특정 조건에서만 사용할 수 있는 매우 빠른 알고리즘
- 데이터의 값을 직접 비교하며 위치를 바꾸는 '비교 기반 정렬'이 아니라, 각 데이터가 몇 번 등장했는지 횟수를 세는 방식의 정렬 알고리즘
- $O(N + K)$라는 압도적으로 빠른 속도
- 데이터의 크기가 한정되어 있고 중복된 값이 많을수록 매우 유리 (예: 시험 성적 정렬)
- **사용 조건**:
  - 데이터 크기 범위가 제한되어 있고 정수 형태일 때만 사용 가능 (실수는 무한범위) 
  - 일반적으로 최댓값과 최솟값의 차이가 1,000,000을 넘지 않을 때 효과적\

```
public class CountingSort {
    public static void main(String[] args) {
        // 모든 원소의 값이 0보다 크거나 같다고 가정
        int[] array = {7, 5, 9, 0, 3, 1, 6, 2, 9, 1, 4, 8, 0, 5, 2};

        // 1. 데이터 중 최댓값을 구함 (리스트의 크기를 결정하기 위함)
        int max = 0;
        for (int i = 0; i < array.length; i++) {
            if (array[i] > max) {
                max = array[i];
            }
        }

        // 2. 모든 범위를 포함하는 배열 선언 및 0으로 초기화
        // 최댓값이 9라면 0~9까지 총 10개의 공간이 필요하므로 (max + 1)
        int[] count = new int[max + 1];

        // 3. 데이터를 하나씩 확인하며 데이터의 값과 동일한 인덱스의 데이터를 1씩 증가
        for (int i = 0; i < array.length; i++) {
            count[array[i]] += 1;
        }

        // 4. 배열에 기록된 정보 확인 및 출력
        System.out.print("정렬 결과: ");
        for (int i = 0; i < count.length; i++) { // 인덱스(데이터 값) 순회
            for (int j = 0; j < count[i]; j++) { // 저장된 횟수만큼 반복 출력
                System.out.print(i + " ");
            }
        }
    }
}
```
- 시간복잡도 : $O(N + K)$
  - $N$은 데이터의 개수, $K$는 데이터 중 최댓값의 크기
  - N개의 데이터를 확인한 후, K크기의 리스트의 값들을 확인해야 하기 때문
- 공간 복잡도: $O(N + K)$
  - 때에 따라 심각한 비효율성을 초래할 수 있습니다.
  - 예를 들어 데이터가 단 2개(0과 999,999)뿐이라도 크기가 1,000,000인 리스트를 만들어야 합니다.
## Java의 정렬 라이브러리
- 주요 정렬 메서드
  - `Arrays.sort()`: 기본 배열(int[], char[] 등)이나 객체 배열을 정렬할 때 사용합니다.
  - `Collections.sort()`: 리스트(ArrayList, LinkedList 등) 자료구조를 정렬할 때 사용합니다
  - 내림차순은 Collections.reverseOrder()을 해야한다 (원시타입은 불가)
- 정렬 방식의 차이
  - Primitive type(int, double 등): Dual-Pivot Quicksort를 사용합니다. 매우 빠르지만 최악의 경우 $O(N^2)$이 될 가능성이 아주 희박하게 존재합니다.
  - Object type(Integer, String, 사용자 정의 객체): 파이썬과 동일하게 Timsort를 사용합니다. 이는 정렬의 안정성(Stable)을 보장하기 위함입니다.
  - 평균 $O(N \log N)$
- 정렬 기준 설정
  - `Comparable`: 클래스 기본 정렬 기준을 설정할 때 사용합니다. (예: 학생 클래스를 번호순으로 기본 설정)
  - `Comparator`: 그때그때 다른 기준으로 정렬하고 싶을 때 사용합니다. (예: 번호순 클래스를 점수순으로 정렬)
```
class Student implements Comparable<Student> {
    int id;
    String name;

    Student(int id, String name) {
        this.id = id;
        this.name = name;
    }

    @Override
    public int compareTo(Student other) {
        // 학번(id) 기준으로 오름차순 정렬
        return this.id - other.id; 
    }
}
```

public class Main{

    static class Student implements Comparable<Student> {
        String name;
        int score;

        Student(String name, int score) {
            this.name = name;
            this.score = score;
        }

        @Override
        public int compareTo(Student o) {
            return Integer.compare(this.score, o.score);
        }

    }

    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        BufferedWriter bw = new BufferedWriter(new OutputStreamWriter(System.out));
        
        
        int N = Integer.parseInt(br.readLine());
        Student[] arr = new Student[N];
        for(int i=0;i<N;i++) {
            StringTokenizer st = new StringTokenizer(br.readLine());
            String name = st.nextToken();
            int score = Integer.parseInt(st.nextToken());
            arr[i] = new Student(name, score);
        }

        Arrays.sort(arr);


        for(int i=0;i<N;i++) {
            bw.write(arr[i].name +" ");
        }

        
        bw.flush();
        bw.close();


    }

    
}
